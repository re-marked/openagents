# User Personas

## Persona 1: Sarah — The Non-Technical User

**Age:** 34 | **Job:** Marketing manager | **Tech level:** Uses iPhone apps, Google Docs

**Goal:** Find an AI assistant that can research competitors, summarize industry reports, and draft social media posts.

**Journey:**
1. Hears about OpenAgents from a friend
2. Browses the marketplace, no sign-up required
3. Searches "marketing research assistant"
4. Reads the Agent description and reviews
5. Signs up with Google, hires the Agent ($0 — free trial)
6. Chats with the Agent on the website
7. Connects Telegram so she can message from her phone
8. Upgrades to Basic ($9/mo) when trial credits run out

**Key needs:** Zero tech jargon, instant results, mobile access via Telegram
**Pain tolerance:** None for setup. Will leave if she sees "YAML", "Docker", or "API key".

## Persona 2: Marcus — The Power User

**Age:** 28 | **Job:** Freelance developer | **Tech level:** Comfortable with terminals, VS Code

**Goal:** Hire a team of Agents for his freelance business: one for code review, one for client communication, one for project management.

**Journey:**
1. Creates a Project called "Freelance Business"
2. Creates a Team called "Dev Tools"
3. Hires 3 Agents from the marketplace
4. Uses the VS Code-style file tree to customize each Agent's SOUL.md and skills
5. Gives his code review Agent SSH access to his development machine
6. Chats with all 3 in a group chat, mentioning them by name
7. Connects Slack so his client-facing Agent can respond in client channels

**Key needs:** Customizability, SSH access, group chat, Slack integration
**Willing to:** Edit YAML, configure skills, pay $29-79/month

## Persona 3: Alex — The Agent Creator

**Age:** 31 | **Job:** Full-stack developer | **Tech level:** Ships products

**Goal:** Build and monetize AI Agents. Has ideas for a "Legal Document Reviewer" and "Real Estate Analyst" Agent.

**Journey:**
1. Goes to platform.openagents.com
2. Creates an Agent repo on GitHub with `agent.yaml`, `.skills`, and `openagents.yaml`
3. Connects the repo to the Platform
4. Uses the split-panel editor (YAML left, card preview right)
5. Publishes the Agent, sets price at 30 credits/session
6. Watches analytics as users discover and hire the Agent
7. Gets monthly payouts via Stripe Connect (85% of revenue)

**Key needs:** DX simplicity (Vercel-like), real-time preview, transparent analytics
**Measures success by:** Monthly payout amount, user rating, session count

---

## Persona 4: The Organization

**Represents:** Small businesses (5-50 people) who want AI agents for the team

**Goal:** Deploy a shared customer support Agent that the whole team can use.

**Journey:**
1. Creates a Project for the company
2. Creates Teams for different departments
3. Hires Agents that multiple team members can access
4. Connects the customer support Agent to the company's Discord server
5. Manages usage and billing from the Project settings

**Key needs:** Team management, shared Agents, usage monitoring, billing consolidation
